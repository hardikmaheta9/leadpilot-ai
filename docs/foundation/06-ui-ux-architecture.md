\# LeadPilot AI – UI/UX Architecture \& Design System



\## 1. Purpose



This document defines the visual and user experience foundation for LeadPilot AI.



The goal is to make the product feel like a premium SaaS platform, not a basic admin panel.



\## 2. Product Design Philosophy



LeadPilot AI should be:



\- Clean

\- Fast

\- Professional

\- Business-focused

\- AI-assisted

\- Easy to navigate

\- Responsive



Every screen should help the user get more clients.



\## 3. Brand Identity



Product Name: LeadPilot AI  

Company: WebApp Infoway  

Version: 0.1 Alpha  

Tagline: Discover. Analyze. Connect. Convert.

\- Page Title: 24px to 30px

\## 10. Button Rules



Allowed button types:



\- Primary

\- Secondary

\- Success

\- Danger

\- Ghost



No random button colors.



\## 11. Table Rules



Every major table should support:



\- Search

\- Filters

\- Pagination

\- Export

\- Bulk actions

\- Status badges

\- Row actions



\## 12. Form Rules



Every form should include:



\- Clear labels

\- Required field indicator

\- Inline validation errors

\- Save button

\- Cancel button

\- Proper spacing

\- Mobile responsiveness



\## 13. Empty State Rules



Every empty screen should guide the user.



Example:



No leads found.



Suggested action:



\- Add manual lead

\- Import CSV

\- Create search keyword



\## 14. AI UX Rules



AI should appear as an assistant, not a replacement.



AI actions:



\- Analyze Company

\- Score Lead

\- Generate Email

\- Generate Proposal

\- Summarize Opportunity

\- Recommend Next Step



AI output should always be reviewable before sending or saving.



\## 15. Responsive Design



Desktop is primary.



Must also support:



\- Tablet

\- Mobile

\- Collapsible sidebar

\- Responsive tables

\- Mobile-friendly forms



\## 16. UX Golden Rule



Every screen must help answer one question:



What should the user do next to win more business?
